Документация проекта

Краткое техническое описание архитектуры и основных маршрутов приложения.

OpenAPI YAML

Frontend

Jinja2-шаблоны, локальный CSS, адаптивные страницы HR и кандидата без зависимости от CDN.

Backend

Flask-приложение с маршрутами авторизации, создания вакансий, прохождения теста и просмотра отчётов.

Данные и AI

PostgreSQL или SQLite fallback. Ollama используется при наличии, иначе работает детерминированная fallback-оценка.

Основной сценарий

  1. HR регистрируется или открывает обзор процесса.
  2. Создаёт вакансию по текстовому описанию требований.
  3. Система выделяет навыки и формирует вопросы.
  4. Кандидат проходит тест по ссылке.
  5. HR открывает итоговый отчёт и сравнивает кандидатов.

Ключевые маршруты

  • /dashboard - панель HR
  • /vacancy/new - создание вакансии
  • /test/<id> - тест кандидата
  • /vacancy/<id>/result - общий отчёт

Таблицы

  • users - HR-пользователи
  • vacancy - вакансии
  • question - вопросы
  • test_session и answer - прохождения и ответы

Обзор процесса

Маршрут /showcase открывает рабочий пример с HR-аккаунтом, вакансией, вопросами и результатами кандидатов.

Пример API-описания

GET /health
POST /signup
POST /login
POST /vacancy/new
GET /test/{vacancy_id}
GET /test/result/{session_id}
